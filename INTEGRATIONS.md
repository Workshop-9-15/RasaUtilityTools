# RasaUtilityTools Integrations Documentation

## Overview

RasaUtilityTools integrates with multiple external systems to provide a complete automated FAQ deployment pipeline. This document covers all integration points, API configurations, and deployment procedures.

## Integration Architecture

```
┌─────────────────┐    ┌──────────────────┐    ┌─────────────────┐
│     GitHub      │───▶│   Jenkins CI/CD  │───▶│    Rasa X       │
│   (Source)      │    │   (Automation)   │    │  (Deployment)   │
└─────────────────┘    └──────────────────┘    └─────────────────┘
         │                       │                       │
         ▼                       ▼                       ▼
┌─────────────────┐    ┌──────────────────┐    ┌─────────────────┐
│ Website Repo    │    │  Model Training  │    │ Production Bot  │
│ (FAQ Updates)   │    │  (Automated)     │    │   (Active)      │
└─────────────────┘    └──────────────────┘    └─────────────────┘
```

## GitHub Integration

### Repository Configuration

#### Webhook Setup
Configure GitHub webhook to trigger Jenkins builds on FAQ changes:

**Webhook Configuration:**
- **Payload URL**: `https://<jenkins-url>/github-webhook/`
- **Content Type**: `application/json`
- **Events**: Push events (specifically for `newfaq.md` changes)
- **Active**: Enabled

#### Branch Strategy
```bash
# Main branch for production FAQ content
main/master

# Feature branches for FAQ updates
feature/update-faq-<topic>

# Automated Jenkins branches
Jenkins-Build-<BUILD_NUMBER>
```

### Authentication Configuration

#### Personal Access Token
Create GitHub Personal Access Token with permissions:
- **repo**: Full repository access
- **workflow**: Update GitHub Actions workflows
- **write:packages**: Package registry access (if needed)

### Automated Pull Request Creation

#### PR Workflow
```bash
# Jenkins pipeline creates automated PRs
git checkout -b "Jenkins-Build-${BUILD_VERSION}"
git remote add upstream <github-repo-url>
git pull upstream master

# Copy generated FAQ content
cp faq.mdx <website-repo-location>/faq.mdx

# Commit and push changes
git add <faq-directory>
git commit -m "Automated FAQ Update - Build ${BUILD_VERSION}"
git push --set-upstream origin Jenkins-Build-${BUILD_VERSION}

# Create pull request using hub CLI
hub pull-request \
  --base <repo>:master \
  --head <username>:Jenkins-Build-${BUILD_VERSION} \
  --message "Automated FAQ Update - Build ${BUILD_VERSION}"
```

#### PR Template
```markdown

- [ ] FAQ content matches source

**Auto-generated by Jenkins CI/CD Pipeline**
```

## Website Integration

### FAQ Content Synchronization

#### Content Format Transformation
```python
# Transform newfaq.md to website format
def generate_web_faq():
    with open("newfaq.md") as source:
        with open("faq.mdx", "w") as target:
            for line in source:
                if line.startswith("### "):
                    # Write question
                    target.write(line)
                elif line.startswith("answer: "):
                    # Write answer
                    answer = line.replace("answer: ", "").strip()
                    target.write(f"{answer}\n\n")
```

from prometheus_client import Counter, Histogram

message_counter = Counter('bot_messages_total', 'Total bot messages')
response_time = Histogram('bot_response_time_seconds', 'Bot response time')

def track_metrics(func):
    def wrapper(*args, **kwargs):
        start_time = time.time()
        result = func(*args, **kwargs)
        response_time.observe(time.time() - start_time)
        message_counter.inc()
        return result
    return wrapper

## Troubleshooting Integration Issues

### Common Problems

#### Authentication Failures
**Problem**: API calls return 401/403 errors  
**Solution**: Verify credentials and token expiry
```bash
# Test authentication
curl -X POST https://<rasa-x-url>/api/auth \
  -H "Content-Type: application/json" \
  -d '{"username": "user", "password": "pass"}'
```

#### Network Connectivity
**Problem**: Services cannot communicate  
**Solution**: Check network configuration and firewall rules
```bash
# Test connectivity
telnet <rasa-x-host> 443
curl -I https://<rasa-x-url>/api/health
```

#### Model Deployment Failures
**Problem**: Models fail to activate in Rasa X  
**Solution**: Check model format and Rasa X logs
```bash
# Verify model structure
tar -tzf models/<model-name>.tar.gz

# Check Rasa X logs
docker logs <rasa-x-container>
```
